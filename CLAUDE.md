# CLAUDE.md

## プロジェクト概要

人間関係タイプ診断アプリ。9問の質問から、「普段のあなた（自然体のあなた）」の関わり方を8タイプで診断する。

- **立ち位置**：診断は特定の場面（職場など）を前提にせず、自然体の対人スタイルそのものを映す。職場・プライベートに共通で使える。文脈（職場向け／大切な人向け）は入口（LP・結果画面）で出し分ける。
- **提供価値（差別化ポイント）**：普段のあなたを知り、それを踏まえたコンサル（読み解き・セッション）を行うことで、職場やプライベートの人間関係の居心地が良くなるきっかけをつくる。診断は「場面ごとの振る舞い」ではなく「自然体のあなた」を測るのが肝。
- **役割**：診断は無料の共通入口かつ共通言語（＝関係を語る語彙）。収益は診断そのものでなく、その先の人的サービス層（ワークショップ・セッション・ガイド）で立てる。詳細な事業設計・ファネルは `ファネル図.md` と参照先スプレッドシートを見る。
- 設問は職場前提の言い回し（会議・タスク等）をあえて排し、場面に依存しない自然体の傾向を測る方針とする。

- `prototype.html` — 検証用のプロトタイプ（スタンドアロンのHTML/CSS/JS）。`?ref=`/`?mode=lead` で紹介者向け（深掘り診断の申込）に切替
- `index.html` / `.nojekyll` — GitHub Pages 公開用。`index.html` はクエリ（`?ref=` 等）を引き継いで `prototype.html` へリダイレクト
- `apps-script/` — 診断結果・検証フィードバック・紹介者リード（メール/自由記述）をGoogleスプレッドシート（回答ログ）に記録する Google Apps Script Web App とセットアップ手順
- `guide-3day-spec.md` — メール登録者に手動で送る「3日間の読み解きガイド」を生成するための仕様。受診者タイプを指示すると、この仕様と `all-types.html` の素材から Day1〜3 を生成する（現状は手動運用、将来は本実装）。

## 表記ルール

- **診断コピー・本文で「——」（ダッシュ）を使用禁止**。ためらい・転換・言い換えは、句点「。」読点「、」や「：」「（）」などで自然につなぐこと。単独の「—」も本文では使わない。

## プレビューURLの提示ルール

- フロントエンド（`prototype.html`）の変更をブランチにプッシュしたら、ユーザーが指示しなくても**毎回**ブランチのプレビューURLを提示する。
- 形式：`https://htmlpreview.github.io/?https://raw.githubusercontent.com/yuki-b4/work-relation-quiz/refs/heads/<ブランチ名>/prototype.html`
- URLはブランチ＋ファイルで**固定**（毎回同じ）。最新の内容はプッシュ後、`raw` のCDNキャッシュ（数分）で反映される。
- バックエンドのみ（`apps-script/` 等）の変更にはプレビューを出さず、再デプロイが必要な旨を伝える。
