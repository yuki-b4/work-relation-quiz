# 文脈出し分け仕様（個人／法人）

診断は職場・プライベートに共通で使える「自然体のあなた」を測る共通入口かつ共通言語。
文脈（個人／法人）は入口（LP・結果画面）で出し分ける。本書はその出し分け素材を確定させたもの。

> 実装時期：LP制作フェーズ（ロードマップ M2〜M3）。現状の `prototype.html` は単一フローのまま（検証用）。
> 実装方式の想定：`?ctx=personal｜corp` で①〜④を出し分け。結果カードの核は共通のまま。
> 本文に長音ダッシュは使わない（プロジェクトの表記ルール）。

## プロダクトコンセプト

- **個人**：自然体で居心地の良い関係を築く
- **法人**：自然体で居心地の良い職場環境を作る

## 出し分けマップ

| 箇所 | 出し分け | 方針 |
| :-- | :-: | :-- |
| ① 入口（INTRO/LP） | 必要 | 文脈がここで決まる。最初かつ最重要の分岐 |
| ② 立場（POSITION） | 必要 | 法人＝職場ロール、個人＝関係の相手 or スキップ |
| ③ CTA（deepBlock） | 必要 | 商品・訴求を文脈で切替 |
| ④ CTA内の具体例（placeholder等） | 必要 | 例示を文脈に合わせる |
| ⑤ FRAME（自然体の説明） | 不要（共通） | 「特定の場面でなく自然体」を測るのが核。文脈化しない |
| ⑥ 設問24問 | 不要（共通） | 場面非依存で確定済み |
| ⑦ 結果カードの核（タイプ・5軸・あるある・強み・ワンポイント・相性） | 不要（共通） | これが共通言語。両文脈で同じものを使う＝差別化の肝 |

## ① 入口（INTRO/LP）

| 要素 | 個人 | 法人 |
| :-- | :-- | :-- |
| eyebrow | 人間関係タイプ診断 | 職場の人間関係タイプ診断 |
| hero | 自然体のあなたを知ることが、居心地のいい関係づくりの第一歩。 | 自然体で尊重し合えるチームが、居心地のいい職場をつくる。 |
| lead | 力を抜いたときの「自然体のあなた」を知り、大切な人との関わりをラクにする。 | メンバー一人ひとりの「自然体」を知り、押し付けでなく自律的に尊重し合えるチームへ。 |

## ② 立場（POSITION）

- **法人**：現状の職場ロール（経営者／管理職／メンバー／業務委託・フリーランス／その他）のまま。
- **個人**：「いちばん関係を良くしたいのは誰ですか？（パートナー・配偶者／家族・親子／友人／職場の人）」に差し替え、またはスキップ。

## ③ CTA（deepBlock）

| 要素 | 個人 | 法人 |
| :-- | :-- | :-- |
| 見出し | たった1人との関係が、毎日の居心地を左右していませんか？ | チームの「関わり方」が、職場の居心地を左右していませんか？ |
| 商品（軽→高） | 3日間の読み解きガイド → 個人マインドセッション | 読み解きワークショップ → エグゼクティブマインドセッション |
| 訴求 | 大切なあの人との関係を、一緒にほどく | チーム全員のタイプを可視化し、自律的に尊重し合う関係をつくる |

## ④ CTA内の具体例（placeholder）

- **個人**：例：パートナーに、つい強く当たってしまって…
- **法人**：例：任せたいのに、つい自分でやってしまう部下がいて…

## 共通（出し分けしない）で確定済みのコピー

- INTRO（現状の単一フロー・`prototype.html`）は個人寄りの中立コピーで確定：
  - hero：自然体のあなたを知ることが、居心地のいい関係づくりの第一歩。
  - lead：力を抜いたときの「自然体のあなた」の関わり方を、8つのタイプで映し出します。自分を知り、身近なあの人を知るほど、関わりはラクになっていきます。
- FRAME・設問・結果カードの核は文脈非依存で共通。
