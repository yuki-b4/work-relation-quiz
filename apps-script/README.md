# 回答収集のセットアップ（Google Apps Script）

`prototype.html` の診断結果を Google スプレッドシートに記録するための、Apps Script Web App のセットアップ手順です。

## 全体像

```
prototype.html  ──fetch(POST)──▶  Apps Script Web App  ──行追加 / 行更新──▶  回答ログ（スプレッドシート）
```

- 回答ログ: [職場の人間関係タイプ診断_回答ログ](https://docs.google.com/spreadsheets/d/1J9sfu2F3uJtpriORSkz7v-5uNQHD6mR4j3UZ22H1Q7M/edit)
  - 記録先タブは `指標v2`（`Code.gs` の `SHEET_NAME`。存在しなければ自動作成）。精度でなくエンゲージメント／拡散を測る新アンケート用で、旧 `20260619~` とは別管理。
- 追加ライブラリ・外部サービスは不要、無料。

## デプロイ手順（1回だけの作業）

1. <https://script.google.com> を開き、**「新しいプロジェクト」** を作成する。
2. デフォルトの `myFunction` を消し、[`Code.gs`](./Code.gs) の中身を **全部コピペ** する。
3. 保存する（プロジェクト名は任意。例：「人間関係診断 回答収集」）。
4. 右上 **「デプロイ」▶「新しいデプロイ」** を選ぶ。
5. 種類の歯車から **「ウェブアプリ」** を選択し、以下を設定：
   - 実行するユーザー：**自分**
   - アクセスできるユーザー：**全員**
6. **「デプロイ」** を押し、初回は **承認** する
   （自分の Google アカウントを選択 →「詳細」→「（プロジェクト名）に移動」→「許可」）。
7. 表示される **「ウェブアプリのURL」** をコピーする
   （`https://script.google.com/macros/s/.../exec` の形式）。

## URLを反映する

コピーした URL を Claude に伝えるか、`prototype.html` の先頭付近にある定数に貼り付けてください。

```js
const SHEET_ENDPOINT = 'ここにウェブアプリのURL';
```

空のままだと送信は行われず、診断の体験だけができます（回答は記録されません）。

## 動作確認

- **エンドポイント単体**：ウェブアプリのURLをブラウザで開くと `{"ok":true,...}` が表示される。
- **通し**：`prototype.html` を開いて診断を最後まで回すと回答ログに1行追加され、検証アンケート（5問）に答えて「回答を送信する」を押すと、その行のアンケート列が埋まる。

## 2つのモード（URLで出し分け）

`prototype.html` は同じファイルのまま、URLパラメータで対象を切り替えます。

- **検証モード（8名＝過去客向け）**：プレーンなURL。検証アンケート5問（これは私だ／他者当てはめ／見せたい・話したい／深掘りしたい／滑った部分）を表示・記録。
- **紹介モード（紹介者向け）**：`?ref=<紹介者コード>` 付きURL（または `?mode=lead`）。深掘り診断の申込（メール＋自由記述）を表示・記録し、`ref` で紹介元（コード）も残す。コードは別タブ `紹介者マスタ` で名前に逆引きする（下記）。
  - 例：`prototype.html?ref=TKtp46k`（田中さん用の紹介リンク）
  - 深掘りの読み解き自体は当面は手動で生成して送付する想定（取得→自動でLLM読み→自動送信はM2〜M3）。

## 紹介者コード → 名前の自動変換（紹介者マスタ）

`?ref=` には紹介者の**名前ではなくコード**（イニシャル＋英数字5文字）を入れる。URLに名前が出ず、ログ側でコード→名前に逆引きする。

**セットアップ（1回だけ・スプレッドシート操作）**

1. 回答ログに新タブ **`紹介者マスタ`** を作成：`A1`＝`コード`、`B1`＝`紹介者名`。2行目以降に「コード｜紹介者名」を記入（例：`TKtp46k｜田中`）。
2. 記録タブ **`指標v2`** の **`AB1`** に `紹介者名`、**`AB2`** に次式を貼る（`C`列＝`紹介元`＝コード。`AB`列に名前が自動表示され、新しい回答行も自動変換）：

   ```
   =ARRAYFORMULA(IF(C2:C="","",IFERROR(VLOOKUP(C2:C,紹介者マスタ!A:B,2,FALSE),"未登録")))
   ```

**コードの形式**：`イニシャル（英大文字）＋ ランダム英数字5文字`（例：田中 → `TK`＋`tp46k`＝`TKtp46k`）。ランダム部はURLから名前を推測されないための不透明化。VLOOKUPは大文字小文字を区別しないので、大小だけ違うコードは作らない。

**使い方**：田中さん用リンク `…/?ref=TKtp46k` から来た回答は `紹介元=TKtp46k` で記録され、`紹介者名` 列に `田中` が自動表示される。未登録コードは `未登録`。

## 記録の流れ（2フェーズ）

回答ID（`回答ID` 列）をキーに、診断とアンケートを同じ行へまとめる。

1. **診断完了時**（`action: create`）：`回答ID` つきの行を追加する。アンケート未回答でも診断結果は取りこぼさない。
2. **アンケート回答時**（`action: update`）：`回答ID` で同じ行を探し、アンケート列だけを埋める。

## 記録される項目

タイムスタンプ／モード／紹介元／9問の回答（極）／3軸の判定（本音・衝突・重心）／タイプコード・タイプ名／検証アンケート5問（これは私だ・他者当てはめ＋誰・見せたい/話したい＋誰・深掘りしたい・滑った部分）／メールアドレス・自由記述（紹介モードのみ）／回答ID。

## 注意

- **コードを修正したら必ず再デプロイ**：「デプロイ ▶ デプロイを管理 ▶ 鉛筆 ▶ バージョン＝新バージョン」で更新しないと反映されません。
- アクセスを「全員」にするため URL は公開扱いです。8名検証では問題ありませんが、不特定多数に広く配る段階になったら合言葉トークン等での保護を追加してください（必要なら対応します）。
